شرح scripts/convert_seminar_to_karmadock.py

مسودة ترجمة تقنية أولية للمراجعة. قد تحتاج المصطلحات والصياغة إلى تنقيح.

الغرض

يحوّل هذا الأمر تخطيط الملفات المسطح للندوة إلى التخطيط المنفصل لكل مركّب الذي تتطلبه المعالجة المسبقة في KarmaDock الأصلي. ينسخ البنى ولا يحولها كيميائيًا، ولذلك لا يتغير سوى المجلدات وأسماء الملفات.

اختبر فهمك

  1. لماذا ينسخ هذا المحوّل البنى بدل تعديل الكيمياء الجزيئية؟

    اعرض الإجابة

    لأن الجزيئات المصدر منقحة بالفعل؛ ولا يحتاج KarmaDock الأصلي إلا إلى تنظيم مختلف للأدلة وأسماء الملفات لكل معقّد.

  2. ما الثابت الذي ينشئه المحوّل لكل معقّد صالح؟

    اعرض الإجابة

    ينشئ <id>/<id>_ligand.sdf و<id>_protein.pdb، وهما الاسمان الدقيقان اللذان تتوقعهما المعالجة المسبقة الأصلية.

  3. لماذا يجب التعامل مع ملف الربيطة وملف البروتين كزوج؟

    اعرض الإجابة

    تحتاج معالجة KarmaDock المسبقة كلا الشريكين الجزيئيين، لذلك لا يستطيع معقّد يملك ملف مصدر واحدًا فقط تكوين مدخل إرساء صالح.

موقعه في المنظومة

تستدعيه run_train.sh وrun_full_train.sh وrun_infer.sh قبل استخراج الجيب وتوليد الرسوم البيانية في الشيفرة الأصلية. يأخذ --csv و--src_dir و--out_dir، ثم يكتب <out_dir>/<id>/<id>_ligand.sdf و<id>_protein.pdb.

اختبر فهمك

  1. لماذا يُستدعى هذا المحوّل قبل استخراج الجيب وتوليد الرسم البياني؟

    اعرض الإجابة

    لأن تلك الخطوات الأصلية تجتاز تنظيم KarmaDock للمدخلات بحسب المعقّد، وهو لا يطابق تنظيم المقرر المسطح للبنى.

  2. كيف تقلل دالة complex_records المشتركة عدم الاتساق في خط الأنابيب؟

    اعرض الإجابة

    تمنح تحويل التدريب والاستدلال المعرّفات وأسماء ملفات المصدر المرتبة نفسها عبر مخططات CSV المدعومة.

  3. ما دور --src_dir و--out_dir؟

    اعرض الإجابة

    يحتوي --src_dir بنى المصدر المسماة وفق المقرر، بينما يستقبل --out_dir شجرة الأدلة المعاد تنظيمها وفق KarmaDock.

شرح الشيفرة

import argparse
import os
import shutil
import sys

from seminar_csv import complex_records

ينسخ shutil.copy2 المحتويات ويحافظ على البيانات الوصفية حيثما أمكن، بينما يضمن مساعد CSV المشترك اتساق المعرّفات.

def main():
    ap = argparse.ArgumentParser()
    ap.add_argument("--csv", required=True, help="seminar CSV (e.g. proto_train.csv)")
    ap.add_argument("--src_dir", required=True, help="dir with the refined .sdf/.pdb files")
    ap.add_argument("--out_dir", required=True, help="output KarmaDock-layout dir")
    args = ap.parse_args()

    for path in (args.csv, args.src_dir):
        if not os.path.exists(path):
            sys.exit(f"ERROR: not found: {path}")

جميع المسارات إلزامية. يُتحقق من المدخلين قبل بدء عمل الخرج، وتطبع sys.exit(string) الخطأ وتنهي التنفيذ بحالة غير صفرية.

    os.makedirs(args.out_dir, exist_ok=True)
    records = complex_records(args.csv)
    ok = miss = 0
    for cid, lig, prot in records:
        s_lig, s_prot = os.path.join(args.src_dir, lig), os.path.join(args.src_dir, prot)
        if not (os.path.exists(s_lig) and os.path.exists(s_prot)):
            miss += 1
            continue

يمكن لإعادة التشغيل استخدام مجلد الخرج نفسه. لا يكون المركّب صالحًا إلا عند وجود الشريكين الجزيئيين؛ وتُحصى الأزواج الناقصة ثم تُتجاوز.

        d = os.path.join(args.out_dir, cid)
        os.makedirs(d, exist_ok=True)
        shutil.copy2(s_lig, os.path.join(d, f"{cid}_ligand.sdf"))
        shutil.copy2(s_prot, os.path.join(d, f"{cid}_protein.pdb"))
        ok += 1
    print(f"converted {ok}/{len(records)} complexes ({miss} missing source files)")


if __name__ == "__main__":
    main()

يحصل كل مركّب ناجح على الأسماء الدقيقة التي تتوقعها الشيفرة الأصلية. يكشف الملخص فقد البيانات من دون جعل بضعة إخفاقات في المعالجة أخطاء قاتلة. يمنع حاجز الوحدة النسخ عند الاستيراد.

اختبر فهمك

  1. لماذا يُتحقق من مساري الإدخال قبل إنشاء دليل المخرجات؟

    اعرض الإجابة

    يفشل البرنامج بوضوح عند غياب CSV أو دليل المصدر قبل أن يترك عملًا جزئيًا في المخرجات.

  2. ماذا يحدث عند غياب أحد الشريكين الجزيئيين لسجل في CSV؟

    اعرض الإجابة

    يزيد المحوّل عدّاد المفقودات، ويتجاوز ذلك المعقّد، ويتابع معالجة الأزواج المكتملة الأخرى.

  3. لماذا يُعد ملخص converted النهائي مهمًا تشغيليًا؟

    اعرض الإجابة

    لأن الأزواج المفقودة لا تؤدي إلى خروج غير صفري، فيكشف الملخص فقد التغطية الصامت الذي يجب تأكيده بعدد الرسوم اللاحق.

تنبيهات وملاحظات

  • لا تسبب الأزواج المفقودة خروجًا غير صفري؛ يجب فحص عدد الرسوم البيانية اللاحق والملخص.
  • تُستبدل الملفات الهدف الموجودة، بينما تبقى مجلدات الخرج القديمة غير المرتبطة.
  • استخراج الجيب وبناء الرسم البياني خطوتان أصليتان لاحقتان وليستا جزءًا من هذا المحول.

اختبر فهمك

  1. لماذا قد يظل خروج العملية الناجح ممثلًا لتحويل غير مكتمل؟

    اعرض الإجابة

    تُتجاوز أزواج الربيطة والبروتين المفقودة من دون إفشال الأمر، لذلك لا تثبت حالة النجاح وحدها اكتمال تغطية مجموعة البيانات.

  2. ما خطر الحالة القديمة الناتج من استخدام exist_ok=True عند إعادة التشغيل؟

    اعرض الإجابة

    تُستبدل ملفات الهدف المتوقعة، لكن الأدلة غير المرتبطة والمتبقية من مجموعات بيانات أقدم تظل في شجرة المخرجات.

  3. لماذا ينبغي ألا يتوقع المستخدمون ملفات رسوم بيانية بعد انتهاء هذا الأمر؟

    اعرض الإجابة

    لأن المحوّل يعيد تنظيم ملفات البنى فقط؛ أما استخراج الجيوب وتوليد الرسوم الأصليان فخطوتان لاحقتان منفصلتان.
