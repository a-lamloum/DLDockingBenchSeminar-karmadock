scripts/convert_seminar_to_karmadock.py

Purpose

This command converts the seminar's flat file layout into the per-complex layout required by upstream KarmaDock preprocessing. It copies rather than chemically transforms structures, so only directory and file naming change.

Check your understanding

  1. Why does this converter copy structures instead of modifying molecular chemistry?

    Show answer

    The source molecules are already refined; upstream KarmaDock needs only a different per-complex directory and filename convention.

  2. What invariant does the converter establish for each usable complex?

    Show answer

    It creates <id>/<id>_ligand.sdf and <id>_protein.pdb, the exact names expected by upstream preprocessing.

  3. Why must ligand and protein files be treated as a pair?

    Show answer

    KarmaDock preprocessing needs both molecular partners, so a complex with only one source file cannot form a usable docking input.

How it fits in

run_train.sh, run_full_train.sh, and run_infer.sh call it before upstream pocket extraction and graph generation. It takes --csv, --src_dir, and --out_dir, then writes <out_dir>/<id>/<id>_ligand.sdf and <id>_protein.pdb.

Check your understanding

  1. Why is this converter called before pocket extraction and graph generation?

    Show answer

    Those upstream steps traverse KarmaDock's per-complex input layout, which does not match the seminar's flat structure layout.

  2. How does the shared complex_records helper reduce pipeline inconsistency?

    Show answer

    It gives training and inference conversion the same ordered IDs and source filenames across supported CSV schemas.

  3. What are the roles of --src_dir and --out_dir?

    Show answer

    --src_dir contains the seminar-named source structures, while --out_dir receives the reorganized KarmaDock directory tree.

def main():
    ap = argparse.ArgumentParser()
    ap.add_argument("--csv", required=True, help="seminar CSV (e.g. proto_train.csv)")
    ap.add_argument("--src_dir", required=True, help="dir with the refined .sdf/.pdb files")
    ap.add_argument("--out_dir", required=True, help="output KarmaDock-layout dir")
    args = ap.parse_args()

    for path in (args.csv, args.src_dir):
        if not os.path.exists(path):
            sys.exit(f"ERROR: not found: {path}")

Check your understanding

  1. Why are both input paths checked before the output directory is created?

    Show answer

    The script fails clearly on a missing CSV or source directory before leaving partial output work behind.

  2. What happens when one molecular partner is absent for a CSV record?

    Show answer

    The converter increments the missing count, skips that complex, and continues processing other complete pairs.

  3. Why is the final converted summary operationally important?

    Show answer

    Missing pairs do not cause a nonzero exit, so the summary exposes silent coverage loss that downstream graph counts must confirm.

Gotchas / notes

  • Missing pairs do not cause a nonzero exit; downstream graph counts and the summary must be checked.
  • Existing target files are overwritten, while unrelated stale output directories are retained.
  • Pocket extraction and graph construction are later upstream steps, not part of this converter.

Check your understanding

  1. Why can a successful process exit still represent incomplete conversion?

    Show answer

    Missing ligand/protein pairs are skipped without making the command fail, so success status alone does not prove full dataset coverage.

  2. What stale-state risk comes from using exist_ok=True on reruns?

    Show answer

    Expected target files are overwritten, but unrelated directories left from older datasets remain in the output tree.

  3. Why should users not expect graph files after this command finishes?

    Show answer

    The converter only reorganizes structure files; upstream pocket extraction and graph generation are separate later steps.
