شرح scripts/run_full_stage2_ddp.sh

مسودة ترجمة تقنية أولية للمراجعة. قد تحتاج المصطلحات والصياغة إلى تنقيح.

الغرض

هذا مشغّل معزول للمرحلة 2 على البيانات الكاملة باستخدام التدريب المتوازي الموزّع (DDP) في PyTorch. يعيد استخدام أوزان المرحلة 1 للقراءة فقط ورسومًا بيانية مبنية مسبقًا، ويكتب في مساحة عمل منفصلة، مما يتيح تجارب متعددة GPU من دون لمس تشغيل التدريب الرئيس.

اختبر فهمك

  1. لماذا يعيد هذا المشغّل استخدام الرسوم المبنية مسبقًا ونقطة تحقق Stage-1 المجمّدة؟

    اعرض الإجابة

    لعزل تجربة Stage-2 متعددة وحدات GPU عن المعالجة المسبقة المكلفة وعن التغييرات في مساحة عمل Stage-1 أو التدريب الكامل الرئيسية.

  2. ما الذي يجعل مخرجات المشغّل معزولة عن مدخلاته المخصصة للقراءة فقط؟

    اعرض الإجابة

    تشير المدخلات افتراضيًا إلى البيانات والرسوم واللقطة الموجودة في مجلد منزل مركّب، بينما تُكتب نقاط التحقق وحالة W&B تحت S2_WORK_DIR منفصل.

  3. لماذا يلائم التدريب المتوازي الموزّع (DDP) هذه المرحلة ذات البيانات الكاملة تحديدًا؟

    اعرض الإجابة

    يمكن لحمل Stage-2 الكبير توزيع عينات منفصلة على وحدات GPU مع مزامنة تدرجات النموذج، ما يقلل زمن التنفيذ مقارنة بعملية واحدة.

موقعه في المنظومة

ينقل condor/full_stage2_mgpu.sub وcondor/smoke_2gpu.sub هذا البرنامج بجوار train_ddp.py وseminar_csv.py. تكون المدخلات افتراضيًا في مسارات ضمن مجلد المنزل المثبت، ويمكن تجاوزها بواسطة FULL_DATA_DIR وFULL_GRAPHS_DIR وS2_WORK_DIR وSTAGE1_CKPT وS2_BATCH وS2_ACCUM وS2_EPOCHS. الخرج هو $S2_WORK_DIR/ckpt_s2_ddp/ إضافة إلى سجلات W&B.

اختبر فهمك

  1. كيف يغيّر full_stage2_mgpu.sub وsmoke_2gpu.sub المشغّل نفسه لخدمة تجربتين مختلفتين؟

    اعرض الإجابة

    يختاران الموارد وتجاوزات البيئة، مثل عدد وحدات GPU وحد الحقبة، مع تجهيز المشغّل ومدرّب DDP نفسيهما.

  2. لماذا تكون متغيرات البيئة مفيدة مع أن البرنامج يوفّر قيمًا افتراضية؟

    اعرض الإجابة

    لأنها تنقل المدخلات والمخرجات الكبيرة المركّبة من المنزل أو تضبط الدفعة والتراكم ونقطة التحقق والحقب من دون تعديل الملف التنفيذي.

  3. ما الأثر المتوقع بعد تشغيل ناجح؟

    اعرض الإجابة

    يُتوقع أفضل نموذج تحقق في $S2_WORK_DIR/ckpt_s2_ddp/karmadock_team002.pkl، مع حالة الاستئناف والتسجيل في مساحة العمل المعزولة نفسها.

شرح الشيفرة

#!/usr/bin/env bash
# ...
set -euo pipefail
set -x

KD=/app/KarmaDock
export PYTHONPATH=/app/KarmaDock:${PYTHONPATH:-}
SUBMIT="$PWD"

يساعد سلوك Bash الصارم وتسجيل التتبع على تشخيص إخفاقات العمليات المتعددة. تصبح نسخة KarmaDock الأصلية قابلة للاستيراد، ويُتوقع وجود الملفات المنقولة في جذر صندوق المهمة.

DATA="${FULL_DATA_DIR:-$HOME/repro_test/full_data}"
GRAPHS="${FULL_GRAPHS_DIR:-$HOME/repro_test/work_full}"
S2WORK="${S2_WORK_DIR:-$HOME/stage2_mgpu_test/work}"
STAGE1_CKPT="${STAGE1_CKPT:-$HOME/stage2_mgpu_test/stage1_snapshot.pkl}"
BS="${S2_BATCH:-8}"; ACC="${S2_ACCUM:-2}"
EP="${S2_EPOCHS:-1000}"

mkdir -p "$S2WORK"
[ -f "$STAGE1_CKPT" ] || { echo "ERROR: Stage-1 checkpoint not found: $STAGE1_CKPT"; exit 3; }

تُبقي القيم الافتراضية المدخلات الكبيرة في مجلد المنزل المثبت، وكل المخرجات في شجرة stage2_mgpu_test المعزولة. تطبع المجموعة { ...; } خطأ واضحًا وتخرج بالحالة 3 إذا غابت نقطة تحقق التهيئة المجمدة.

NGPU=$(nvidia-smi -L 2>/dev/null | wc -l); [ "${NGPU:-0}" -lt 1 ] && NGPU=1
RUN="full_stage2_ddp_$(date +%Y%m%d-%H%M%S)"
echo "=== Stage-2 DDP on $NGPU GPU(s); effective batch = $((NGPU*BS*ACC)); run=$RUN ==="

يطبع nvidia-smi -L سطرًا لكل GPU مرئي. يعرض التعبير الحسابي حجم الدفعة الفعلي العالمي بوصفه عدد العمليات × الدفعة الفيزيائية لكل عملية × خطوات التجميع. يجعل الطابع الزمني اسم W&B الأساسي المقروء فريدًا. لا ينشئ الرجوع إلى قيمة واحدة GPU؛ بل يختار فرع العملية الواحدة ويترك train_ddp.py يقرر بين CUDA وCPU.

ARGS=( "$SUBMIT/train_ddp.py"
  --csv "$DATA/full_train.csv"   --graph_dir "$GRAPHS/full_train/graphs" --complex_dir "$GRAPHS/full_train/complex"
  --val_csv "$DATA/full_val.csv" --val_graph_dir "$GRAPHS/full_val/graphs"
  --out_dir "$S2WORK/ckpt_s2_ddp" --init_model "$STAGE1_CKPT" --pos_r 1
  --lr 1e-4 --weight_decay 1e-4 --patience 20 --epochs "$EP"
  --batch_size "$BS" --accum_steps "$ACC" --num_workers 3 --random_seed 42 --resume
  --wandb --wandb_project karmadock-seminar --wandb_run_name "$RUN" )

تحافظ المصفوفة على حدود الوسائط بأمان. تفعّل pos_r=1 التدريب المشترك للإرساء وRMSD وتسجيل MDN. تستخدم هذه النسخة الأصلية weight_decay بقيمة 1e-4 وpatience بقيمة 20. وتحافظ مدخلات التدريب والتحقق الصريحة على التقسيم المنقح.

if [ "$NGPU" -gt 1 ]; then
  torchrun --standalone --nnodes=1 --nproc_per_node="$NGPU" "${ARGS[@]}"
else
  python3 -u "${ARGS[@]}"
fi
echo "=== Stage-2 DDP done: $S2WORK/ckpt_s2_ddp/karmadock_team002.pkl ==="

يبدأ torchrun عملية Python واحدة لكل GPU، ويمرر RANK وLOCAL_RANK وWORLD_SIZE؛ ويستخدمها train_ddp.py لتهيئة NCCL وDDP. ينشئ --standalone --nnodes=1 نقطة الالتقاء محليًا على جهاز واحد. يتجنب تخصيص GPU واحد إعداد مجموعة العمليات ويستخدم مسار العملية الواحدة في البرنامج.

اختبر فهمك

  1. كيف تُحسب الدفعة الفعّالة العامة المعروضة؟

    اعرض الإجابة

    تُحسب بضرب عدد عمليات GPU المرئية في الدفعة الفيزيائية لكل عملية S2_BATCH وفي عدد نوافذ التراكم S2_ACCUM.

  2. لماذا تهم --pos_r 1 ومسارات التحقق الصريحة في Stage 2؟

    اعرض الإجابة

    تفعّل pos_r=1 تعلّم الإحداثيات والتقييم معًا، بينما تحفظ رسوم التحقق الكاملة المنفصلة تقسيم التدريب والتحقق المنتقى.

  3. ما الذي يتغير عندما تكون وحدة GPU واحدة فقط مرئية؟

    اعرض الإجابة

    يستدعي المشغّل python3 العادي بدل torchrun، فيتجنب إعداد مجموعة العمليات مع الاحتفاظ بوسائط المدرّب نفسها.

تنبيهات وملاحظات

  • يُستخدم nvidia-smi داخل برنامج يفعّل pipefail. إذا غاب الأمر أو أخفق بدل أن يعرض صفر أجهزة، فقد ينهي خط الإسناد البرنامج قبل تنفيذ الرجوع الاحتياطي.
  • يعيد --resume استخدام $S2WORK/ckpt_s2_ddp/last.pt؛ لذلك قد يتابع استخدام مجلد الخرج نفسه تشغيلًا سابقًا رغم امتلاك RUN طابعًا زمنيًا جديدًا.
  • تعطي القيم الافتراضية BS=8 وACC=2 دفعة عالمية 64 فقط مع أربع وحدات GPU؛ ويلزم تجاوز ملف الإرسال لأعداد أخرى.

اختبر فهمك

  1. لماذا قد لا يعمل رجوع عدّ وحدات GPU مطلقًا إذا فشلت nvidia-smi نفسها؟

    اعرض الإجابة

    لأن العد ينتجه خط أنابيب داخل برنامج يستخدم pipefail، وقد ينهي فشل nvidia-smi التنفيذ قبل الوصول إلى رجوع العد الصفري اللاحق.

  2. كيف يمكن لاسم تشغيل W&B جديد أن يستأنف حالة محسّن قديمة؟

    اعرض الإجابة

    يغيّر الطابع الزمني الاسم المعروض، لكن --resume يقرأ last.pt من دليل نقطة التحقق المعاد استخدامه بصورة مستقلة عن ذلك الاسم.

  3. لماذا تعيد إعدادات الدفعة الافتراضية إنتاج دفعة فعّالة مقدارها 64 على أربع وحدات GPU فقط؟

    اعرض الإجابة

    القيم الافتراضية هي دفعة 8 وتراكم 2، لذلك لا يبلغ الناتج 64 إلا عندما تكون world_size مساوية لأربعة؛ وتحتاج التخصيصات الأخرى إلى تجاوزات.
